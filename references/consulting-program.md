# Foresight Consulting Program — Full Engagement Guide

## Overview

A professional foresight engagement for a consulting firm is not a one-shot analysis.
It is a structured research program delivered in phases, with defined inputs, outputs,
client touchpoints, and quality checkpoints at each stage.

This guide covers the complete lifecycle: from client brief to final report.

---

## Phase 0 — Scoping & Framing

**Duration:** 1–2 weeks
**Objective:** Align on the question, the horizon, the audience, and the budget.

### Client Brief Template

Ask these questions before designing anything:

**On the question:**
- What is the primary question this foresight study must answer?
- Who will use the results, and for what decision?
- Are there sub-questions or secondary topics?
- Are any futures explicitly out of scope (political, too speculative)?

**On the horizon:**
- What time horizon is relevant for the client's decision cycle?
- Short (1–5 yr) / Medium (5–15 yr) / Long (15–30 yr)?
- Is the horizon fixed or can it be discussed?

**On the system:**
- What is the system boundary? (sector, geography, technology, institution)
- What is clearly inside vs. outside the system?
- Are there adjacent systems that must be considered?

**On stakeholders:**
- Who are the key internal stakeholders? (sponsor, users, skeptics)
- Are there external experts or partners to involve?
- Is there a validation committee or scientific board?

**On constraints:**
- Total budget and number of consulting days?
- Fixed deadlines (board presentation, strategic planning cycle)?
- Confidentiality requirements on sources or outputs?
- Preferred deliverable formats (slide deck, report, workshop)?

### Scoping Output: Study Design Document (1–2 pages)

Deliver before starting research:
- Confirmed research question
- Selected methodology (with rationale)
- Phase plan with milestones and deliverables
- Expert panel composition
- Risk register (data gaps, access issues, timeline risks)
- Validation process

## Phase 1 — Environmental Scan

**Duration:** 2–4 weeks
**Objective:** Map the landscape of trends, signals, and uncertainties.

### Research Protocol

#### Source Typology (prioritize in this order)

| Source type | Examples | What it provides |
|---|---|---|
| **Scientific literature** | Google Scholar, Scopus, Web of Science | Validated trends, emerging research fronts |
| **Think tank reports** | RAND, McKinsey GI, Brookings, OECD, WEF | Synthesized megatrends, policy implications |
| **Institutional foresight** | NIC Global Trends, EU foresight, CGSP | Authoritative long-term scenarios |
| **Expert interviews** | Targeted panel (see below) | Tacit knowledge, early signals, divergent views |
| **Patent databases** | USPTO, EPO, WIPO | Technology emergence and trajectories |
| **Media monitoring** | Factiva, Meltwater, RSS + manual | Weak signals, emerging discourse |
| **Grey literature** | NGO reports, startup pitches, conference proceedings | Emerging issues before mainstream |
| **Field observation** | Site visits, ethnography, user research | Ground truth vs. desk research |

#### Signal Classification Grid

For each signal or trend captured, classify along two axes:

```
High impact
    │  MONITOR          │  ACT
    │  (watch closely)  │  (prioritize)
    │                   │
────┼───────────────────┼────── Certainty
    │                   │  High
    │  IGNORE           │  INCLUDE IN
    │  (low relevance)  │  BASELINE
    │                   │
Low impact
         Low certainty ←→ High certainty
```

| Field | Description |
|---|---|
| **Signal ID** | Unique reference |
| **Description** | What is happening, where, since when |
| **Sources** | At least 2 independent sources |
| **Type** | Megatrend / Emerging trend / Weak signal / Wild card |
| **STEEP domain** | S / T / E / E / P |
| **Impact level** | High / Medium / Low (for this system) |
| **Certainty** | High / Medium / Low |
| **Horizon** | When might this become significant? |
| **Implications** | First-order effects on the study system |

#### Expert Panel Design

**Composition principles:**
- 12–20 experts for a focused study; 30–50 for a broad one
- Mandatory diversity: disciplinary, geographic, generational, gender
- Include at least 20% "outliers" — people who hold unconventional views
- Avoid groupthink: no more than 30% from the same sector

**Recruitment process:**
1. Draft a long list (3× target size)
2. Screen for genuine expertise + availability + independence
3. Confirm participation with a clear brief on time commitment
4. Send pre-reading material before any interview or workshop

**Interview guide (semi-structured, 45–60 min):**
- Opening: "What are the 2–3 most significant changes you see in [domain] over [horizon]?"
- Trends: "Which trends do you consider certain? Which keep you up at night?"
- Weak signals: "What are people in your field talking about that hasn't gone mainstream yet?"
- Wild cards: "What low-probability event would change everything if it happened?"
- Scenarios: "If you had to describe 3 very different futures for [domain], what would they look like?"
- Closing: "What question should we be asking that we haven't asked?"

### Phase 1 Deliverable: Environmental Scan Report

Structure:
1. Executive summary (1 page)
2. Methodology note
3. Megatrends (5–8) — each with evidence base and implications
4. Emerging trends (10–15) — each with maturity stage (Molitor curve)
5. Weak signals catalogue (20–40) — classified grid
6. Wild cards register (5–10) — with activation scenarios
7. Key uncertainties — what we don't know and why it matters
8. Appendix: source bibliography, expert list (if not confidential)

## Phase 2 — Structural Analysis

**Duration:** 1–3 weeks
**Objective:** Identify the key variables and actors that will determine future states.

### Workflow

**Step 1 — Variable identification**
- Extract variables from the environmental scan
- Workshop with client team to validate and enrich (2–3 hours)
- Target: 20–40 variables for MICMAC, 5–8 for scenario axes

**Step 2 — Choose the right structural tool**

| Situation | Tool |
|---|---|
| Complex system with many interdependencies | MICMAC (full matrix) |
| Multiple actors with conflicting interests | MACTOR |
| Need to explore all possible combinations | Morphological analysis |
| Need to prioritize 2 key uncertainties | Schwartz axes |
| Need stakeholder buy-in through participation | Facilitated variable mapping |

**Step 3 — MICMAC (if applicable)**
→ See `references/godet-lipsor.md` for full protocol

Output: classification of variables into motrices / relay / dependent / excluded
→ Select relay variables as scenario drivers

**Step 4 — MACTOR (if applicable)**
→ See `references/godet-lipsor.md` for full protocol

Output: actor-objective matrix, alliance/conflict map, strategic recommendations

### Phase 2 Deliverable: Structural Analysis Note (3–5 pages)

- Variable map with classification
- Key drivers identified (3–5 variables that will shape the future)
- Actor map (if relevant)
- Rationale for scenario construction approach

## Phase 3 — Scenario Construction

**Duration:** 2–4 weeks
**Objective:** Build 3–4 contrasted, coherent, plausible futures.

### Scenario Design Process

**Step 1 — Choose the scenario architecture**

| Method | When to use |
|---|---|
| **2-axis / Shell** | Clear dominant uncertainties, need 4 quadrants |
| **French morphological** | Many components, need to explore combinations |
| **Three Horizons** | Transition focus, H3 vision needed |
| **Narrative-first** | Creative, participatory, or cultural foresight |

**Step 2 — Formulate hypotheses**
For each key driver (from Phase 2), define 2–3 contrasted hypotheses:
- H1: optimistic / cooperative
- H2: status quo / muddling through
- H3: pessimistic / conflictual
- H4 (optional): rupture / wild card activated

**Step 3 — Build scenario skeletons**
Select coherent combinations of hypotheses.
Apply the 4C test to each skeleton:
- **Coherent** — no internal contradiction
- **Credible** — plausible given current trends
- **Clear** — understandable by non-experts
- **Consequential** — useful for decisions

**Step 4 — Write scenario narratives**

Each scenario narrative must include:

| Element | Description |
|---|---|
| **Title** | Evocative, memorable — often a metaphor |
| **Opening image** | A vivid description of the world at the horizon |
| **Pathway** | Key events and decisions that led from now to then |
| **Actors** | Who won, who lost, who adapted, who disappeared |
| **Economy** | What the economic logic of this world looks like |
| **Society** | Daily life, values, tensions |
| **Technology** | What technologies dominate, what failed |
| **Governance** | Who holds power and how |
| **Closing image** | What a day in this world feels like |
| **Implications** | For the client's sector, organization, or decision |

**Recommended length:** 400–800 words per scenario narrative.
**Tone:** Present tense, written as if the future is now.

**Step 5 — Scenario validation workshop**

Run a half-day workshop with the client team:
- Present the 4 scenario skeletons (without narratives)
- Ask: "Is anything missing? Is anything impossible? What would you add?"
- Refine before writing final narratives
- After narratives: "Which world worries you most? Which excites you? Why?"

### Phase 3 Deliverable: Scenario Report (8–15 pages)

1. Introduction: why these scenarios, what they cover
2. Scenario architecture: axes, drivers, construction logic
3. Four scenario narratives (full text)
4. Comparative table: key dimensions × scenarios
5. Wild card annex: 3–5 low-probability disruptors and how they would affect the scenarios

## Phase 4 — Strategic Implications

**Duration:** 1–2 weeks
**Objective:** Translate futures into decisions, priorities, and actions.

### Tools

**Wind Tunneling**
Test the client's current strategy against each scenario:
- Is the strategy viable in this future?
- What does it not anticipate?
- What new risks emerge?
- What new opportunities appear?

Output: Strategy Robustness Matrix

| Strategy element | Scenario A | Scenario B | Scenario C | Scenario D |
|---|---|---|---|---|
| Current positioning | Robust ✓ | Fragile ⚠️ | Collapse ✗ | Robust ✓ |
| Revenue model | Robust ✓ | Robust ✓ | Fragile ⚠️ | Collapse ✗ |
| Key partnerships | Fragile ⚠️ | Robust ✓ | Robust ✓ | Fragile ⚠️ |

**Backcasting**
For the preferred or most challenging scenario:
1. Define the target state at horizon
2. Work backwards: what must be true 5 years before?
3. And 2 years before that?
4. What decisions must be made now to enable that path?

Output: Decision Roadmap with milestones and trigger points

**Early Warning System**
Define 3–5 indicators per scenario to monitor:
- What observable signals would confirm we are moving toward Scenario X?
- At what threshold should the organization adjust its strategy?

### Phase 4 Deliverable: Strategic Implications Report (5–10 pages)

1. Strategy robustness assessment
2. Scenario-specific risks and opportunities
3. Robust strategies (work in all scenarios)
4. Conditional strategies (bet on specific scenarios)
5. Decision roadmap with milestones
6. Early warning indicator dashboard

## Phase 5 — Communication & Transfer

**Duration:** 1–2 weeks
**Objective:** Ensure the work lives beyond the report — in decisions, culture, capability.

### Deliverable Stack by Audience

| Audience | Format | Length | Key content |
|---|---|---|---|
| **Board / C-suite** | Executive briefing | 10–15 slides | 3 scenarios, 5 strategic implications, 3 decisions needed now |
| **Strategy team** | Full report | 30–50 pages | All phases, full scenario narratives, roadmap |
| **Middle management** | Workshop facilitation | Half day | Scenario exploration + team-level implications |
| **External / Public** | Summary publication | 8–12 pages | Edited scenarios, key trends, no confidential content |

### Executive Presentation Structure (15 slides)

1. Why we did this study (context and question)
2. How we did it (method in plain language — 1 slide)
3. The landscape: 5 megatrends shaping the future
4. The key uncertainties: what we don't know
5–8. The four scenarios (1 slide each — title + 5 bullet points)
9. Comparative scenario table
10. What keeps us up at night (wild cards)
11. Your current strategy: where it works, where it doesn't
12. What is robust across all futures
13. What requires a strategic bet
14. Decision roadmap: what to decide now vs. later
15. Early warning: signals to monitor

### Knowledge Transfer

At the end of the engagement, deliver:
- **Methodology note** — so the client can replicate the process internally
- **Signal database** — the full classified scan, not just the highlights
- **Scenario cards** — one-page summary per scenario for internal circulation
- **Workshop facilitation guide** — so the client can run their own future workshops

## Program Governance

### Meeting Cadence

| Meeting | Frequency | Duration | Participants | Purpose |
|---|---|---|---|---|
| Steering committee | Monthly | 1–2 hours | Sponsor + lead consultant | Progress, decisions, risks |
| Working group | Bi-weekly | 1–2 hours | Client team + consultants | Content review, feedback |
| Expert sessions | As needed | 45–60 min | Individual experts | Interviews, validation |
| Workshops | Per phase | Half/full day | Broader stakeholders | Co-construction, validation |

### Quality Assurance

**Internal review (consulting firm):**
- Methodological audit at end of Phase 2 (are the right tools being used?)
- Peer review of scenario narratives (coherence, credibility, contrast)
- Senior partner sign-off before any client presentation

**External validation:**
- At least 2 expert reviewers per major deliverable
- Client validation workshop before final report

### Risk Register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Scope creep | High | Medium | Fixed scope document signed at Phase 0 |
| Low expert engagement | Medium | High | Over-recruit panel; prepare async alternatives |
| Client team not available | Medium | High | Name a dedicated client contact at kickoff |
| Results too abstract | Medium | High | Wind tunneling in Phase 4; concrete roadmap |
| Political sensitivity of scenarios | Low | High | Pre-validate with sponsor before broad sharing |

## Program Timeline Templates

### Compact engagement (6–8 weeks, 15–25 consulting days)

```
Week 1–2   Phase 0 + Phase 1 (parallel scoping + desk research)
Week 3     Phase 2 (structural analysis + client workshop)
Week 4–5   Phase 3 (scenario construction + validation workshop)
Week 6     Phase 4 (strategic implications)
Week 7–8   Phase 5 (communication + transfer)
```

### Standard engagement (3–4 months, 30–50 consulting days)

```
Month 1    Phase 0 + Phase 1 (full scan + expert interviews)
Month 2    Phase 2 + Phase 3 (structural analysis + scenario draft)
Month 3    Phase 3 validation + Phase 4 (strategic implications)
Month 4    Phase 5 (communication, workshops, transfer)
```

### Comprehensive engagement (6–12 months, 60–120 consulting days)

```
Months 1–2   Phase 0 + Phase 1 (full Delphi + field research)
Months 3–4   Phase 2 (full MICMAC + MACTOR)
Months 4–6   Phase 3 (morphological analysis + extensive workshops)
Months 6–8   Phase 4 (wind tunneling + backcasting roadmap)
Months 9–12  Phase 5 (rollout, training, internal capability building)
```

## Mode: Program Design

When a user asks Claude to design a foresight program or study for a client:

1. **Ask the Phase 0 questions** (brief template above) — do not skip this
2. **Produce a Study Design Document** before any analysis
3. **Select the right timeline template** based on budget/duration constraints
4. **Identify which reference files to use** per phase:
   - Phase 1 → `anglosaxon-tools.md` (scanning), `delphi-protocol.md` (expert panel)
   - Phase 2 → `godet-lipsor.md` (MICMAC, MACTOR)
   - Phase 3 → `godet-lipsor.md` (scenarios), `cla-guide.md` (narrative depth)
   - Phase 3+4 → `anglosaxon-tools.md` (Wind Tunneling, Backcasting)
   - Phase 4+5 → `nic-global-trends.md` (executive communication model)
5. **Produce deliverables progressively** — do not skip to conclusions
