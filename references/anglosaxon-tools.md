# Outils Anglo-Saxons — Cone of Plausibility, Three Horizons, Emerging Issues

## 1. Cone of Plausibility (Joseph Voros, 2001)

### Principe
Le Cone of Plausibility est le cadre visuel de référence des futures studies modernes.
Il représente l'espace des futurs comme un cône qui s'élargit avec le temps, et organise
les futurs en 5 catégories imbriquées.

### Les 5 catégories de futurs

```
                           ┌─────────────────┐
                           │   PRÉFÉRABLE    │  ← ce que nous voulons
                      ┌────┴────────────────┐│
                      │     PLAUSIBLE      ││  ← cohérent avec notre
                 ┌────┴───────────────────┐││    compréhension du monde
                 │       PROBABLE        │││  ← extension des tendances
            ┌───┴──────────────────────┐││││
            │        POSSIBLE         ││││││  ← techniquement réalisable
      ┌─────┴─────────────────────────┘│││││
      │         PRÉVU                  │││││  ← business as usual
      └──────────────────────────────────────
PRÉSENT ─────────────────────────────► FUTUR
```

| Catégorie | Définition |
|---|---|
| **Prévu (Projected)** | Continuation du passé — "business as usual" |
| **Possible** | Tout ce qui pourrait théoriquement arriver (lois physiques respectées) |
| **Plausible** | Ce qui pourrait arriver compte tenu de notre compréhension actuelle |
| **Probable** | Ce qui a de bonnes chances d'arriver si les tendances se poursuivent |
| **Préférable** | Ce que nous voulons qu'il arrive (dimension normative, subjective) |

### Usage en pratique
- Utilisé en **introduction** pour cadrer l'espace de la réflexion prospective
- Permet de positionner les scénarios : tendanciel (probable), alternatifs (plausibles),
  rupture (possible), vision (préférable)
- Rend explicite que le "probable" n'est pas le seul futur légitime à explorer

### Lien avec le triptique de Jouvenel
- Jouvenel : possible / probable / souhaitable
- Voros : possible / plausible / probable / préférable
- Voros ajoute la distinction *plausible* (cohérent avec notre modèle du monde) vs
  *possible* (techniquement réalisable mais hors modèle) — distinction analytiquement précieuse

## 2. Three Horizons (Bill Sharpe / International Futures Forum, 2010)

### Principe
Le cadre Three Horizons (3H) cartographie la coexistence de trois logiques temporelles
simultanément présentes dans le système actuel. Ce n'est pas un cadre prédictif —
c'est un cadre de **lecture du présent** pour identifier les forces de transformation.

### Les trois horizons

**H1 — Le présent dominant**
- Institutions, pratiques, modèles économiques qui "fonctionnent" aujourd'hui
- En déclin à long terme mais encore puissants et légitimes
- Résistent au changement car ils représentent des intérêts établis
- *Exemple :* le modèle de la voiture individuelle thermique

**H2 — La zone de tension / transition**
- Innovations, expérimentations, mouvements en cours qui ne sont ni H1 ni H3
- Certaines renforcent H1 (innovations incrémentales), d'autres préfigurent H3
- Zone d'ambiguïté et de conflit — c'est là que se jouent les transitions
- *Exemple :* les véhicules hybrides, le covoiturage, les ZFE

**H3 — Les futurs émergents**
- Signaux faibles, mouvements marginaux, visions alternatives de long terme
- Peu visibles aujourd'hui mais porteurs d'un futur radicalement différent
- La question est : lesquels de ces signaux H3 vont s'amplifier ?
- *Exemple :* la ville sans voiture, la mobilité comme service public universel

### Comment l'utiliser
1. Identifier les éléments H1 du système (ce qui est dominant aujourd'hui)
2. Repérer les tensions H2 (ce qui est en train de changer)
3. Explorer les signaux H3 (ce qui pourrait définir le futur)
4. Analyser lesquels des éléments H2 renforcent H1 vs. préfigurent H3
5. Formuler une vision H3 et travailler en backcasting vers le présent

### Combinaison avec le Backcasting
Three Horizons identifie *où on veut aller* (H3) ; le backcasting trace *comment y aller*.

## 3. Emerging Issues Analysis (Graham Molitor)

### Principe
Toute transformation majeure commence par un signe faible, marginal, souvent ignoré.
L'Emerging Issues Analysis (EIA) modélise la **courbe de maturité** d'un enjeu émergent,
de son émergence marginale jusqu'à son intégration dans le mainstream politique et médiatique.

### La courbe de maturité d'un enjeu

```
Visibilité
publique
  ↑
  │                                          ████ Legislation
  │                                     █████
  │                               ██████       ████ Media mainstream
  │                          █████
  │                    ███████                       ████ Politique
  │              ███████
  │        ██████                                         ████ Science
  │  ██████
  │██  Penseurs marginaux
  └──────────────────────────────────────────────────► Temps
     Idée    Débat   Recherche  Politique   Loi    Norme
```

### Les 5 stades d'un enjeu émergent

| Stade | Acteurs dominants | Signal |
|---|---|---|
| **Émergence** | Visionnaires, penseurs marginaux, artistes | Articles confidentiels, œuvres d'art, discours alternatifs |
| **Débat précoce** | Académiques, ONG, blogueurs influents | Articles scientifiques, rapports de think tanks |
| **Montée médiatique** | Journalistes, médias spécialisés | Presse grand public, documentaires |
| **Appropriation politique** | Partis politiques, régulateurs | Propositions de loi, consultations publiques |
| **Institutionnalisation** | Gouvernements, entreprises | Lois, normes, standards industriels |

### Application pratique
- **Veille** : surveiller les stades 1-2 pour anticiper les enjeux de demain
- **Décision** : agir en stade 2-3 est optimal — assez précoce pour se positionner,
  assez tardif pour ne pas surinvestir dans un faux signal
- **Calibration des signaux faibles** : distinguer les signaux au stade 1 (spéculatif)
  des signaux au stade 2-3 (probable) — et ajuster le langage en conséquence

### Exemples historiques
- L'effondrement écologique : signal faible dans les années 1960 (Rachel Carson, *Silent Spring*),
  devenu norme législative dans les années 1990-2000
- La privacy numérique : signal faible 2000-2010, institutionnalisé avec le RGPD (2018)
- L'IA générative : signal faible 2015-2020, mainstream depuis 2022-2023 — nous sommes en stade 4

## 4. Wind Tunneling

### Principe
Tester une **stratégie existante** (ou un plan, une politique, une décision) en la
"projetant" dans plusieurs scénarios futurs pour évaluer sa robustesse.

### Processus
1. Définir la stratégie ou décision à tester
2. Construire 3-4 scénarios contrastés (ou utiliser des scénarios existants)
3. Pour chaque scénario : *"Cette stratégie serait-elle efficace dans ce contexte ?"*
4. Identifier les **conditions de succès** et les **points de vulnérabilité**
5. Adapter la stratégie pour la rendre robuste à plusieurs futurs (stratégie "all-weather")

### Critères d'évaluation par scénario
- Faisabilité : la stratégie est-elle exécutable dans ce futur ?
- Pertinence : répond-elle toujours aux besoins dans ce contexte ?
- Risques : quels nouveaux risques ce scénario crée-t-il pour la stratégie ?
- Opportunités : ce scénario ouvre-t-il de nouvelles options ?

### Output
Une **matrice stratégie × scénarios** qui montre clairement quelles décisions sont
robustes (fonctionnent dans tous les scénarios) vs. fragiles (dépendent d'un futur spécifique).
